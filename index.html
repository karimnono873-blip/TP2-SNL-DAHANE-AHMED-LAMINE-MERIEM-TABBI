TP N°2 - Plan de Phase, Boucles Algébriques & Oscillateur de Van der Pol

1. Modélisation et Résolution de Systèmes Couplés

La première partie du TP exige la modélisation d'un système dynamique décrit par deux équations différentielles non linéaires fortement couplées. Les équations initiales (1) et (2) sont données sous forme implicite :

(1) ÿ₁ + 2*ẏ₁*ẏ₂ + y₁ = u₁ (2) ÿ₂ + 3*ẏ₁*ẏ₂ + 4*y₂ + 2*y₁*y₂ = u₂

Pour implémenter ce système dans un environnement de schéma bloc causal comme Simulink, nous devons isoler les termes dérivés de plus haut degré. En réarrangeant algébriquement, nous obtenons la forme d'état explicite nécessaire à l'intégration numérique :

ẏ₁ = -2*(ẏ₁*ẏ₂) - y₁ + u₁ ẏ₂ = -3*(ẏ₁*ẏ₂) - 4*y₂ - 2*(y₁*y₂) + u₂

Le Phénomène de Boucle Algébrique : En observant ces équations, on constate que ẏ₁ et ẏ₂ dépendent d'eux-mêmes instantanément (présence à gauche et à droite de l'égalité). Dans Simulink, cela force le solveur temporel à utiliser des méthodes itératives (comme Newton-Raphson) à chaque pas de calcul pour résoudre f(z) = 0. Le schéma ci-dessous illustre comment prélever ces signaux de dérivées avant l'intégration pour construire les termes multiplicatifs.

L'Oscillateur de Van der Pol

Ce système représente un circuit électronique à tube à vide avec une résistance non linéaire. Son équation différentielle fondamentale est :

ẍ + μ(x² - 1)ẋ + x = 0

Transformation d'État : Pour intégrer numériquement ce système, nous devons le réduire à un système d'ordre 1. Posons x₁ = x (position) et x₂ = ẋ (vitesse) :

  • ẋ₁ = x₂
  • ẋ₂ = -μ(x₁² - 1)x₂ - x₁

Le paramètre μ (mu) dicte la nature de l'amortissement. S'il est positif, le système tend vers un attracteur stable unique appelé Cycle Limite, caractéristique clé du plan de phase non linéaire.

Moteur Physique (Runge-Kutta RK4)

Friction Non-Linéaire (μ) 0.5
Pos. Initiale (x₁) 1.0
Vit. Initiale (x₂) 1.0

Script MATLAB Officiel - Résolution Multi-Cas

L'implémentation algorithmique MATLAB ci-dessous satisfait rigoureusement le cahier des charges du TP en balayant les matrices de conditions initiales CI ∈ {(-3,3), (3,-3), (1,1)} et les paramètres de friction μ ∈ {-0.5, 0, 0.5}.

% =========================================================================
% INSTITUTION : U.S.T.H.B / F.G.E. - Département d'Automatique
% MODULE      : Systèmes non linéaires (2025-2026)
% SUJET       : TP N° 2 - Plan de Phase - L'oscillateur de Van der Pol
% =========================================================================

function tp2_vanderpol_solver()
    % 1. Configuration des paramètres de simulation
    tspan = [0, 25]; % Horizon temporel d'intégration (en secondes)
    
    % 2. Définition des jeux de paramètres (Selon Cahier des Charges)
    matrice_CI = [-3, 3; 3, -3; 1, 1]; % [x1_initial, x2_initial]
    vecteur_mu = [-0.5, 0, 0.5];       % Paramètre de non-linéarité

    % 3. Préparation de l'interface graphique MATLAB
    figure('Name', 'Analyse du Plan de Phase - Van der Pol', 'Color', 'w', 'Position', [100, 100, 1200, 400]);

    % 4. Boucle d'itération principale sur le paramètre mu
    for idx_mu = 1:length(vecteur_mu)
        mu = vecteur_mu(idx_mu);
        
        % Création d'un sous-graphique pour chaque valeur de mu
        subplot(1, 3, idx_mu);
        hold on; grid on;
        
        % Formatage du graphique
        title(sprintf('Dynamique pour \\mu = %.1f', mu), 'FontWeight', 'bold', 'FontSize', 12);
        xlabel('Variable d''état x_1 (Position)');
        ylabel('Variable d''état x_2 (Vitesse)');
        
        % 5. Sous-boucle pour tester toutes les conditions initiales pour un mu donné
        for idx_ci = 1:size(matrice_CI, 1)
            x0 = matrice_CI(idx_ci, :);
            
            % 6. Déclaration de la fonction d'état du système (modèle mathématique)
            % Forme matricielle implicite: dx/dt = f(t,x)
            sys_vdp = @(t, x) [
                x(2); 
                -mu * (x(1)^2 - 1) * x(2) - x(1)
            ];
            
            % 7. Appel au solveur numérique (Runge-Kutta d'ordre 4/5 à pas variable)
            % ode45 gère automatiquement la taille du pas de calcul dt pour garantir la précision
            [vect_t, matrice_x] = ode45(sys_vdp, tspan, x0);
            
            % 8. Rendu de la trajectoire dans l'espace des phases (x1 vs x2)
            plot(matrice_x(:,1), matrice_x(:,2), 'LineWidth', 1.5, ...
                 'DisplayName', sprintf('CI = (%.1f,  %.1f)', x0(1), x0(2)));
                 
            % Placement d'un marqueur visuel pour indiquer le point de départ de l'intégration
            plot(x0(1), x0(2), 'ko', 'MarkerFaceColor', 'k', 'HandleVisibility', 'off');
        end
        
        % Activation de la légende pour identifier les courbes
        legend('Location', 'best');
        axis([-4 4 -4 4]); % Fixation des axes pour comparaison homogène
    end
    
    % Titre global de l'interface
    sgtitle('Analyse Globale du Plan de Phase - Oscillateur Non Linéaire', 'FontSize', 16);
end
